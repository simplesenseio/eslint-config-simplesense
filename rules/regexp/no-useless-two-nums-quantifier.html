# regexp/no-useless-two-nums-quantifier

disallow unnecessary {n,m} quantifier

  • ⚙️ This rule is included in "plugin:regexp/recommended".
  • 🔧 The --fix option on the command line (opens new window) can automatically fix some of the problems reported by this rule.

# 📖 Rule Details

This rule reports unnecessary {n,m} quantifiers.

/* eslint regexp/no-useless-two-nums-quantifier: "error" */

/* ✓ GOOD */
var foo = /a{0,1}/;
var foo = /a{1,5}/;
var foo = /a{1,}/;
var foo = /a{2}/;

/* ✗ BAD */
var foo = /a{0,0}/;
var foo = /a{1,1}/;
var foo = /a{2,2}/;

# 🔧 Options

Nothing.

# 🚀 Version

This rule was introduced in eslint-plugin-regexp v0.1.0
