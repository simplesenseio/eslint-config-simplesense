# Prefer RegExp#test() over String#match() and RegExp#exec()

✅ This rule is part of the recommended (opens new window) config.

🔧 This rule is auto-fixable (opens new window).

When you want to know whether a pattern is found in a string, use RegExp#test() (opens new window) instead of String#match() (opens new window) and RegExp#exec() (opens new window).

# Fail

if (string.match(/unicorn/)) {}

if (/unicorn/.exec(string)) {}

<template>
	<div v-if="/unicorn/.exec(string)">Vue</div>
</template>

# Pass

if (/unicorn/.test(string)) {}

<template>
	<div v-if="/unicorn/.test(string)">Vue</div>
</template>
