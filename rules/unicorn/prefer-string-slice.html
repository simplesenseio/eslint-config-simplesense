# Prefer String#slice() over String#substr() and String#substring()

💼 This rule is enabled in the ✅ recommended config (opens new window).

🔧 This rule is automatically fixable by the --fix CLI option (opens new window).

String#substr() (opens new window) and String#substring() (opens new window) are the two lesser known legacy ways to slice a string. It's better to use String#slice() (opens new window) as it's a more popular option with clearer behavior that has a consistent Array counterpart (opens new window).

# Fail

foo.substr(start, length);
foo.substring(indexStart, indexEnd);

# Pass

foo.slice(beginIndex, endIndex);
